# 浏览器插件与油猴脚本：我的效率工具流

后来逐渐有一个比较明确的体感：

> 人与人之间的效率差距，不只是“能力”，很大一部分来自工具链。

尤其是长期高频使用电脑的人，最后都会形成一套稳定的“数字工作流”：

- 浏览器使用习惯
- 快捷键体系
- 脚本自动化
- 信息获取路径
- 常用工具组合

这些东西不会写在简历里，但实际影响非常大。

---

# 我最开始对效率工具的理解

早期对插件的印象比较简单：

```text
插件 = 花里胡哨的功能
```

但实际使用一段时间后会变成另一种认知：

> 很多“重复操作”，本质上都是可以被消除的。

比如：

- 少点几十次鼠标
- 少复制粘贴几次文本
- 少切换几次页面
- 少做重复筛选

单次看起来不明显，但长期累积差异会被放大。

---

# 浏览器在现代环境中的角色变化

随着使用场景变化，浏览器逐渐变成核心入口：

- 信息检索
- 视频与内容消费
- 文档编辑
- 代码托管
- AI 工具
- 各类 Web 应用

于是一个很自然的结论是：

> 浏览器本身更像“第二个操作系统”。

插件体系的意义，也随之变成：

> 对整个工作环境的扩展。

---

# 插件的本质：改变网页行为

早期误解是插件只是“增加功能”。

后来逐渐明确一点：

> 插件真正的能力，是修改已有网站的行为逻辑。

例如：

- 改 UI
- 改请求行为
- 改数据展示方式
- 增强交互能力
- 过滤信息噪音

本质上是在做“二次改造”。

---

# 一些长期高频使用的网站增强

## 内容平台类

### Twitter / X

常见增强方向：

- 时间线优化
- 图片与视频增强
- 广告过滤
- 翻译增强

原生体验在信息密度较高时并不稳定，因此增强需求明显。

---

### Pixiv

典型增强点：

- 搜索增强
- 标签优化
- 收藏与管理增强
- 排行榜辅助
- 图片查看体验优化

这类站点插件生态相对成熟。

---

### Bilibili

常见增强：

- 页面净化
- 视频下载
- 自动连播控制
- 评论区优化
- 弹幕相关增强

---

### YouTube

主要问题集中在：

- 广告
- 推荐流
- 字幕与翻译
- 页面结构复杂

因此增强工具覆盖面较广。

---

# PotPlayer：本地播放工具的极端形态

在本地媒体播放场景中，PotPlayer 是一个比较典型的“全功能型工具”。

能力包括：

- 多格式支持
- 字幕控制
- 硬解/软解切换
- 视频滤镜与渲染
- 倍速与截图
- 流媒体播放支持

它的特点不是“某一项特别强”，而是：

> 几乎没有明显短板。

---

# 油猴脚本：网页行为级别的修改能力

Tampermonkey（油猴）带来的变化比较直接：

> 浏览器从“使用工具”变成“可编程环境”。

本质是：

```text
网页加载 → 脚本注入 → DOM / 行为修改
```

---

# 最基础的脚本结构

```javascript
// ==UserScript==
// @match https://example.com/*
// ==/UserScript==

document.body.style.background = "black";
```

最直观的变化是：

> 页面行为确实被改变了。

---

# 油猴脚本的核心能力

本质可以归类为三类：

## 1. DOM 修改

- 改页面结构
- 改样式
- 增强 UI

---

## 2. 行为拦截

- 按钮点击逻辑修改
- 页面事件控制
- 功能屏蔽或增强

---

## 3. 数据处理

- 提取内容
- 清洗信息
- 重组展示

---

# 一些常见脚本用途

- 自动展开内容
- 页面净化（去广告/去冗余模块）
- 自动点击与流程简化
- 页面翻译增强
- 数据抓取辅助

这些脚本通常不复杂，但非常贴近实际使用场景。

---

# 自动化的价值判断标准

后来逐渐形成一个简单判断方式：

> 如果某个操作“重复出现”，它就值得被自动化。

核心不是复杂度，而是频率。

---

# 使用工具过程中遇到的问题

## 1. 插件权限过大

部分插件会请求：

```text
读取所有网站数据
```

本质意味着：

- 页面内容可访问
- 输入数据可读取
- Cookie 可能被访问

因此插件来源变得重要。

---

## 2. 插件维护问题

常见情况：

- 网站结构变更
- API 调整
- 作者停止维护

导致脚本失效。

---

## 3. 网站对脚本的限制增强

随着前端复杂化：

- CSP 策略
- Shadow DOM
- React / Vue 渲染结构
- 请求加密

导致脚本稳定性下降。

---

# 对现代前端复杂度的直观感受

早期页面结构是：

```html
<div>content</div>
```

后期则变成：

- 构建工具链（Webpack / Vite）
- 组件化框架（React / Vue）
- 虚拟 DOM
- 状态管理体系
- 服务端渲染与 hydration

页面本身不再“静态”。

---

# AI 工具带来的新一轮插件生态

在 AI 普及后，浏览器插件再次扩展出新方向：

- Prompt 管理
- 对话导出
- 多模型切换
- 页面内容总结
- 写作辅助

插件逐渐从“网页增强”扩展到：

> “工作流增强”。

---

# 工具流的副作用

工具增强并不总是正向收益，也会带来一些问题：

- 配置时间过长
- 工具碎片化
- 频繁切换环境
- 折腾成本过高

有时会出现：

```text
工具没用多少，配置花了一小时
```

---

# 对工具链的最终理解

后期逐渐形成一个更稳定的判断：

> 工具的价值不在于数量，而在于稳定性与低摩擦。

理想状态是：

- 不需要频繁调整
- 不打断思路
- 长期可用
- 贴合习惯

---

# 回头来看这一阶段

浏览器插件、油猴脚本、播放器工具，本质上并不是“高级技术”。

但它们确实带来一个关键变化：

> 从被动使用工具，转向主动改造使用环境。

而这类习惯，会长期影响后续对自动化、系统设计以及工程效率的理解方式。
