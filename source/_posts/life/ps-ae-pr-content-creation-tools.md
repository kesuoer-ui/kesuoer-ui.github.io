````markdown
# PS、AE、PR：我常用的内容创作工具整理

以前一直觉得：

> “做内容的人才需要 Adobe 全家桶。”

后来发现根本不是。

程序员做项目、写博客、搞二创、做演示、剪视频、做 UI、生成宣传图，最后都会绕回：

- Photoshop
- After Effects
- Premiere

尤其做个人项目时。

没人帮你做图。
没人帮你剪视频。
没人帮你做封面。

最后只能自己全干。

---

# 三件套分别是干什么的

| 软件                | 核心用途                |
| ------------------- | ----------------------- |
| Photoshop（PS）     | 修图 / 绘图 / UI / 合成 |
| After Effects（AE） | 动效 / 特效 / 包装      |
| Premiere（PR）      | 视频剪辑                |

简单理解：

```txt
PS 做静态
AE 做动态
PR 做视频
```
````

---

# Photoshop：几乎万能的图像处理器

PS 真的是个很离谱的软件。

因为它：

- 能修图
- 能画图
- 能抠图
- 能做 UI
- 能做网页切片
- 能调色
- 能合成
- 能做封面
- 能做海报

甚至很多人拿它做：

- 表情包
- Live2D 切图
- 视频封面
- 二创图

---

# 我最常用的几个功能

---

# 局部换色

这是最实用的功能之一。

比如：

- 衣服换色
- 头发换色
- UI 改色
- 武器改色

---

# 最核心的是：

## 不要直接“涂颜色”

新人特别容易：

```txt
画笔 -> 硬涂
```

结果直接塑料感。

---

# 正确做法一般是：

## 选区 + 调整图层

比如：

```txt
色相/饱和度
可选颜色
曲线
色彩平衡
```

---

# 为什么？

因为：

## 调整图层保留了原始明暗结构

也就是：

- 高光
- 阴影
- 材质
- 纹理

不会丢。

---

# 比如衣服换色

流程一般：

```txt
钢笔/快速选择 -> 建选区
↓
羽化边缘
↓
色相饱和度
↓
调整色相
↓
蒙版微调
```

---

# 这样改色会自然很多

因为：

## 原图的光影关系还在

这点特别重要。

---

# 高频低频磨皮

以前觉得：

```txt
美颜 = 一键滤镜
```

后来发现真正高级的修图：

## 都在保细节。

---

# 高频低频本质

把：

- 纹理
- 颜色

拆开处理。

---

# 低频

负责：

- 肤色
- 明暗
- 过渡

---

# 高频

负责：

- 毛孔
- 纹理
- 细节

---

# 于是：

可以只修肤色。

保留真实皮肤纹理。

---

# 这是很多“高级感修图”的来源

否则会：

```txt
磨成塑料娃娃
```

---

# 内容感知填充

PS 后来越来越 AI 化。

尤其：

## 内容识别

非常恐怖。

---

# 比如：

- 去路人
- 去水印
- 去电线
- 去杂物

很多时候：

```txt
框一下 -> 删除
```

就没了。

---

# 老图修复与超分

后来 AI 修图起来后。

很多老二次元图终于有救了。

---

# 典型流程

## 第一步：超分

常见：

- ESRGAN
- Real-ESRGAN
- waifu2x

---

# 第二步：PS 精修

AI 超分的问题：

- 糊
- 假细节
- 边缘炸裂
- 眼睛崩坏

所以最后还得：

```txt
PS手修
```

---

# 我很常干的一件事

## 修头发边缘

因为 AI 很容易：

```txt
头发融化
```

---

# 还有眼睛

眼睛最容易：

```txt
左右不对称
```

必须手修。

---

# AE：最强动效工具

如果说：

```txt
PS 是静态之王
```

那：

```txt
AE 就是动态之王
```

---

# AE 本质是什么？

很多人第一次打开 AE：

```txt
满屏时间轴
```

直接懵。

---

# 实际上：

AE 本质是：

## 图层动画软件

不是传统“剪视频”。

---

# PR 更像：

```txt
拼接视频
```

---

# AE 更像：

```txt
让元素动起来
```

---

# 我最常用的功能

---

# 关键帧

AE 的灵魂。

---

# 所有动画本质都是：

```txt
开始状态
↓
结束状态
↓
中间自动补间
```

---

# 比如：

```txt
透明度：0 -> 100
位置：左 -> 右
缩放：50% -> 100%
```

---

# 缓动（Ease）

这是 AE 动画“高级感”的核心。

新人动画最大问题：

```txt
太机械
```

---

# 因为默认：

```txt
匀速运动
```

看起来像 PPT。

---

# 正常世界里：

物体不会瞬间加速。

所以：

```txt
缓入
缓出
```

很重要。

---

# AE 的经典口诀：

```txt
不会缓动 = 动画没灵魂
```

---

# 遮罩与转场

AE 特别适合：

- MV
- AMV
- 二创
- UI动画

---

# 最经典的：

## 蒙版转场

比如：

- 圆形扩散
- 线条切换
- 遮挡切场

---

# 本质：

```txt
遮罩动画
```

---

# 粒子特效

AE 粒子系统非常强。

比如：

- 雪
- 火花
- 光点
- 漂浮物

---

# 二创里特别常见

尤其：

```txt
氛围感粒子
```

---

# 还有发光

AE 经典：

```txt
Glow
```

---

# 这个东西非常容易：

```txt
一不小心土到爆炸
```

---

# 所以后来学会了一件事：

## 特效不要堆

很多高级感：

其实来自：

```txt
克制
```

---

# PR：真正干活最多的软件

AE 很炫。

但：

## PR 才是真正天天用的

因为：

- 剪视频
- 配字幕
- 配音轨
- 导出
- 压制

全在这里。

# 我最常用的几个功能

---

# 时间线剪辑

核心中的核心。

---

# 最常干的事：

```txt
切空白
切废话
切停顿
```

---

# 一个视频：

剪辑节奏真的很重要。

---

# 节奏差的视频会：

```txt
又臭又长
```

---

# 音轨处理

PR 的音频处理其实很强。

包括：

- 降噪
- 压限
- 混响
- 音量均衡

---

# 特别是：

## 背景音乐压人声

新人很容易：

```txt
BGM炸穿耳朵
```

---

# 正常做法一般：

人声：

```txt
-3db ~ -6db
```

BGM：

```txt
-20db 左右
```

根据内容动态调。

---

# 自动字幕

现在 PR 字幕比以前强很多。

但我后来还是更习惯：

- 剪映
- Whisper
- CapCut

自动识别。

---

# 因为：

## Adobe 中文识别以前真的一般

---

# 代理剪辑

这个是很多人第一次接触 4K 视频时会踩的大坑。

---

# 为什么 PR 卡？

因为：

```txt
H264/H265 解码巨吃CPU
```

尤其：

- 高码率
- 10bit
- 4K

---

# 代理剪辑本质

先生成：

```txt
低清代理文件
```

剪的时候不卡。

导出时：

再替换回原视频。

---

# 这是大项目非常重要的流程

否则：

```txt
时间线一卡一卡
```

心态爆炸。

---

# 关于 Adobe 全家桶的一些真实体验

---

# 优点

## 生态真的太强

尤其：

```txt
PS -> AE -> PR
```

联动。

---

# 比如：

PS 做图。

AE 做动画。

PR 直接导入。

---

# 动态链接（Dynamic Link）

非常离谱。

---

# 缺点

## 吃配置

尤其：

- AE
- PR

非常能吃：

- CPU
- 内存
- 硬盘

---

# AE 尤其离谱

经常：

```txt
缓存几十GB
```

---

# 所以后来我养成了习惯

## 单独准备缓存盘

最好：

```txt
NVME SSD
```

---

# 不然：

```txt
预览卡成PPT
```

---

# Adobe 软件另一个大坑

## 项目文件极其脆弱

尤其 AE。

---

# 经常会：

```txt
昨天还能打开
今天直接损坏
```

---

# 所以后来我学会：

## 疯狂 Ctrl+S

以及：

```txt
project_v1
project_v2
project_final
project_final2
project_final_真的最终版
```

---

# 现在回头看

这些工具最有价值的地方其实不是“会不会用”。

而是：

## 会不会拆解视觉效果

比如看到一个视频：

能意识到：

- 这个是遮罩
- 这个是关键帧
- 这个是曲线缓动
- 这个是粒子
- 这个是辉光
- 这个是调色 LUT

这种“分析能力”反而更重要。

---

# 很多时候技术和创作其实是互通的

尤其后来做：

- Web 前端
- UI 动效

- AI 生成内容
- 二创

会发现：

## 审美、节奏、结构感

真的会互相影响。

包括：

- 前端动画为什么舒服
- 页面布局为什么顺眼
- 视频节奏为什么不拖
- 色彩为什么协调

很多底层感觉其实是共通的。
