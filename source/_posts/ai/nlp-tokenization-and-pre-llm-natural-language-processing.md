# NLP、分词与大模型之前的自然语言处理

现在聊 NLP，很容易默认：

```text id="n1"
ChatGPT
Transformer
LLM
```

但实际上：

> NLP 在大模型出现前，已经发展了非常久。

而且很多今天的大模型概念，本质上都是从传统 NLP 演化来的。

---

# NLP 到底在做什么？

NLP：

# Natural Language Processing

自然语言处理。

核心目标其实很朴素：

> “让机器理解人类语言”

例如：

- 分词
- 情感分析
- 机器翻译
- 文本分类
- 问答
- 语音识别

都属于 NLP。

---

# 中文 NLP 为什么特别麻烦

因为中文：

> 没有空格。

英文：

```text id="n2"
I love machine learning
```

天然分词。

---

# 中文：

```text id="n3"
我喜欢机器学习
```

机器根本不知道：

```text id="n4"
机器学习
```

是不是一个词。

---

# 所以中文 NLP 第一关就是：

# 分词。

---

# jieba 为什么火了很多年

因为：

- 简单
- 好用
- 纯Python
- 效果够用

很多项目都在用。

---

# 最经典的代码

```python id="py2"
import jieba

text = "我喜欢机器学习"

print(list(jieba.cut(text)))
```

输出：

```python id="py3"
['我', '喜欢', '机器学习']
```

---

# jieba 背后其实没那么简单

很多人会以为：

> “就是查词典”

实际上不是。

它结合了：

- Trie树
- DAG
- 动态规划
- HMM

---

# DAG 是什么？

即：

# 有向无环图

例如一句话：

```text id="n5"
研究生命起源
```

可能有多种切法：

```text id="n6"
研究 / 生命 / 起源
研究生 / 命 / 起源
```

于是：

> jieba 会构建所有可能路径。

然后：

```text id="n7"
动态规划找最优解
```

---

# HMM 为什么重要

因为词典永远不够。

尤其：

- 网络新词
- 人名
- 二次元词汇
- 游戏术语

大量不存在词典。

于是：

# Hidden Markov Model

隐马尔可夫模型。

用于：

> 猜测未知词。

---

# NLP 很长时间都在“特征工程时代”

即：

> 人工设计特征。

例如：

- TF-IDF
- N-Gram
- 词频
- 规则匹配

---

# 那时的 NLP 很“规则驱动”

比如聊天机器人：

```text id="n8"
if 包含“天气”
→ 调天气API
```

很多系统其实非常死板。

---

# 后来 Word2Vec 改变了一切

这是 NLP 非常重要的一步。

因为：

> 词终于可以向量化了。

---

# 什么叫词向量？

以前：

```text id="n9"
猫
狗
飞机
```

只是字符串。

模型无法理解关系。

---

# Word2Vec 后

词会变成：

```text id="n10"
[0.123, -0.53, ...]
```

向量。

于是：

- 相似词距离接近
- 语义开始可计算

---

# 最经典例子

```text id="n11"
King - Man + Woman ≈ Queen
```

这就是：

> 语义空间。

---

# 后来为什么 RNN 火了

因为 NLP 最大问题：

# 顺序。

语言有上下文。

例如：

```text id="n12"
苹果很好吃
苹果发布新手机
```

同一个词：

意思完全不同。

---

# RNN 的核心思想

即：

> “前文会影响后文”

模型有记忆。

---

# 但 RNN 很快暴露问题

尤其：

- 长文本
- 梯度消失
- 训练慢

于是后来：

- LSTM
- GRU

开始流行。

---

# 真正改变世界的是 Transformer

2017 年。

论文：

# Attention Is All You Need

直接改变 NLP。

---

# Transformer 最核心思想

即：

# Attention

模型不再按顺序硬记。

而是：

> 动态关注重要部分。

---

# 为什么这东西这么恐怖

因为：

- 并行能力强
- 长文本能力强
- 表达能力极强

于是：

- BERT
- GPT
- T5

全部开始出现。

---

# Tokenizer 是什么？

现在 LLM 时代天天提。

本质其实就是：

# 更高级分词。

---

# 为什么 LLM 不直接按汉字处理？

因为：

- 词频差异
- 稀有词问题
- 多语言问题

于是：

# BPE

Byte Pair Encoding。

开始流行。

---

# BPE 的核心思路

即：

> 高频组合自动合并。

例如：

```text id="n13"
Chat + GPT
```

可能变成一个 token。

---

# 所以现在的大模型其实并不“懂字”

它本质上：

> 在处理 token 序列。

---

# 我第一次真正理解 NLP

其实是做 VITS 的时候。

因为当时：

- 日文
- 中文
- 英文

发音问题很多。

---

# 很多早期 TTS 很依赖分词

尤其：

- 拼音
- 音节
- 停顿
- 重音

都需要 NLP 处理。

---

# 中文 TTS 为什么容易怪

因为中文：

- 多音字太多
- 断句复杂
- 语义依赖强

例如：

```text id="n14"
长大
银行行长
```

发音完全不同。

---

# 后来大模型为什么能“看起来懂语言”

因为：

- 数据量极大
- 参数量极大
- Attention能力极强

于是：

> 模型开始具备统计意义上的语言能力。

---

# 但它本质仍然是概率模型

这一点其实一直没变。

---

# 现在再回头看传统 NLP

会发现很多东西并没有过时。

例如：

- tokenizer
- 分词
- embedding
- attention

今天依然存在。

---

# LLM 并不是凭空出现的

它其实是：

```text id="n15"
规则系统
→ 统计模型
→ 深度学习
→ Transformer
→ LLM
```

一路演化过来的。

---

# 最后

NLP 是 AI 里一个跨度极大的领域。

从：

- 规则匹配
- 中文分词
- HMM
- CRF

一路发展到：

- BERT
- GPT
- 多模态大模型

整个技术路线变化非常剧烈。

但很多核心问题其实一直没变：

> “机器到底如何理解语言”

直到今天，这件事其实也还远远没有真正解决。
