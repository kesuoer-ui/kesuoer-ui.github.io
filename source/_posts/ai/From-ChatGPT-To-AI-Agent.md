# 从 ChatGPT 到 AI Agent：我的大模型认知变化

## 前言

如果把互联网的发展理解为“信息获取效率”的不断提升，那么大模型（LLM）带来的变化，本质上是在把“人类自然语言”直接变成新的操作系统。

搜索引擎时代，人需要适应机器。

AI 大模型时代，机器开始适应人。

而我对这件事的认知变化，大概横跨了这几年整个 AI 行业的发展过程。

---

# 一、最开始：把 ChatGPT 当“会聊天的搜索引擎”

大概在 2020～2021 年前后，OpenAI 的 GPT 系列开始逐渐出圈。

那时候国内其实已经有不少技术圈的人在讨论 GPT-3 了，但普通人真正开始接触，还是在 ChatGPT 爆火之后。

最初体验其实很魔幻。

因为第一次发现：

- 机器真的能理解长串自然语言
- 不需要关键词搜索
- 不需要研究搜索语法
- 可以像真人一样对话
- 还能理解上下文

这和传统搜索引擎完全不是一个时代的东西。

以前搜索：

```text
python aiohttp websocket reconnect
```

后来变成：

```text
我有一个 websocket 长连接，aiohttp 下断线后怎么自动重连？
```

体验差距是跨时代的。

---

## 当时最大的几个问题

但早期 GPT 的问题也非常明显。

### 1. 没有联网能力

早期模型知识库是静态的。

很多版本的数据甚至停留在 2019 年。

于是会出现：

- 新技术不知道
- 新框架不知道
- 时效新闻不知道
- 冷门领域知识很差

本质上：

它只是“预训练后的概率机器”。

不是实时互联网。

---

### 2. 数学和逻辑能力很差

这是很多人早期吐槽 GPT 的地方。

尤其：

- 多步推理
- 复杂数学
- 长逻辑链
- 代码上下文理解

经常会：

- 自信胡说八道
- 陷入错误循环
- 一本正经输出错误答案

后来大家把这种现象称为：

> AI 幻觉（Hallucination）

这也是很多传统工程师最开始不信 AI 的原因。

---

### 3. 强限制与“政治正确”

很多内容会被系统规则限制。

包括：

- 敏感话题
- 暴力内容
- 灰产方向
- 社会争议内容

甚至很多时候：

模型其实“知道”，但不能说。

当时很多人觉得：

> “AI 不够自由”

但后来行业慢慢意识到：

如果完全不限制，风险会非常恐怖。

因为 LLM 的本质：

已经接近“自然语言通用接口”。

它不仅能聊天。

还可能控制工具、生成代码、操作系统。

---

# 二、最早的大模型应用：角色扮演与娱乐

因为早期能力有限。

所以大多数人最开始玩 GPT：

其实是在玩“角色扮演”。

例如：

- AI 女友
- 二次元角色
- 小说续写
- 剧情模拟
- 语言陪聊

这也是 Character AI 一类产品早期爆火的原因。

因为当时大家第一次意识到：

> AI 的情绪反馈已经开始接近真人。

虽然逻辑不稳定。

但“语言风格”已经非常像人。

---

# 三、真正的转折点：模型能力突然开始“起飞”

后来 AI 行业的发展速度开始越来越离谱。

甚至有种：

> “第四次工业革命突然加速”的感觉。

尤其是 GPT-3.5 到 GPT-4 这段时间。

整个行业认知几乎被重构。

---

## 变化最大的几个能力

---

## 1. 自然语言理解能力暴涨

这是最核心的变化。

模型开始真正理解：

- 上下文
- 语气
- 意图
- 方言
- 模糊表达
- 人类习惯

你甚至不用“规范表达”。

它也知道你想干什么。

比如：

```text
帮我看看这段代码为什么抽风
```

模型真的能理解。

这意味着：

> 人类第一次不需要学习机器语言了。

---

## 2. 联网能力出现

这几乎是革命性的。

因为 AI 不再是：

“训练截止日期前的百科全书”。

而是：

“实时互联网助手”。

开始能：

- 搜索网页
- 总结信息
- 获取最新资料
- 查询 API 文档
- 阅读 GitHub
- 整理新闻

这时候很多人开始意识到：

搜索引擎的交互方式可能会被重构。

---

## 3. 多模态能力出现

AI 不再只能处理文本。

开始支持：

- 图片
- 语音
- 视频
- OCR
- 屏幕理解
- 摄像头

后来甚至发展到：

- 实时语音对话
- 视频分析
- GUI 操作
- 屏幕控制

AI 开始越来越像“数字人”。

---

## 4. 长上下文与长期记忆

早期模型最大的问题之一：

上下文太短。

聊着聊着就失忆。

后来：

- 长上下文窗口
- RAG
- Memory
- 向量数据库

开始普及。

AI 终于开始具备：

“持续理解用户”的能力。

---

# 四、AI 从聊天工具变成生产力工具

真正改变世界的，不是“聊天”。

而是：

> 大模型开始接管生产流程。

尤其程序员行业变化最大。

---

# 五、Copilot 时代：程序员第一次真正被 AI 冲击

最早大规模落地的领域：

就是代码生成。

典型代表：

- GitHub Copilot
- Cursor
- Claude Code
- Windsurf
- Continue
- Cline

那时候很多工程师第一次意识到：

AI 不是“代码补全”。

而是在：

- 理解需求
- 生成业务代码
- 修 Bug
- 重构
- 写 SQL
- 生成测试
- 阅读项目
- 分析调用链

虽然经常犯病。

但它已经开始：

> 节约大量重复劳动。

后来很多人调侃：

```text
程序员不会被 AI 替代，
但会被会用 AI 的程序员替代。
```

这句话其实越来越接近现实。

---

# 六、Prompt 工程：第一次出现“自然语言编程”

AI 时代一个非常特殊的概念：

Prompt Engineering（提示词工程）。

这是很多人第一次意识到：

> 自然语言本身也可以成为编程语言。

你怎么描述需求。

会直接影响 AI 输出。

于是开始出现：

- Prompt 模板
- Few-shot
- Chain of Thought
- Role Prompt
- System Prompt

后来甚至出现：

- Prompt 黑话
- 越狱
- 提示词注入攻击
- AI 安全

这时候 AI 已经不再只是“聊天”。

而是开始形成：

新的工程体系。

---

# 七、AI Agent：真正的“数字员工”

后来行业真正的大方向开始出现：

## Agent（智能体）

这也是我认为 AI 真正开始改变行业结构的阶段。

因为：

LLM 本身其实不会“行动”。

它只能“生成文本”。

于是行业开始思考：

> 如果让 AI 调用工具会怎么样？

于是出现：

- Tool Calling
- Function Calling
- Workflow
- MCP
- Browser Use
- Computer Use

AI 开始：

- 调 API
- 控制浏览器
- 操作文件
- 使用数据库
- 调用 Python
- 操作终端
- 自动执行任务

这时候 AI 已经非常接近：

“数字员工”。

---

# 八、LangChain 与 LangGraph：AI 工程化开始形成

随着 Agent 爆发。

行业开始需要：

- 工作流编排
- 状态管理
- 记忆系统
- 工具管理
- 多 Agent 协作

于是：

- LangChain
- LangGraph
- AutoGen
- CrewAI
- Dify
- Flowise

这一类框架开始流行。

企业也开始真正落地：

- 私人知识库
- 企业 AI 助手
- 客服 Agent
- 数据分析 Agent
- 自动化办公 Agent

于是 AI 行业开始出现大量岗位：

- AI 应用开发
- Prompt Engineer
- AI Workflow 工程师
- RAG 工程师
- Agent 工程师

行业进入：

> “AI 工程化”时代。

---

# 九、RAG：AI 为什么突然“懂你公司业务了”

企业真正开始落地 AI 后。

很快发现：

大模型有一个致命问题。

## 不懂企业私有数据。

于是：

RAG（检索增强生成）出现了。

本质流程：

```text
用户问题
→ 检索知识库
→ 找到相关内容
→ 拼接上下文
→ 喂给大模型
→ 生成回答
```

于是 AI 开始：

- 理解企业文档
- 学习内部 SOP
- 查询数据库
- 阅读 API 文档
- 理解业务流程

很多公司第一次意识到：

> AI 终于能进入生产环境了。

---

# 十、AI 行业最大的变化：技术门槛被重新定义

以前的软件行业：

拼的是：

- 代码能力
- 算法能力
- 底层理解

现在依然重要。

但 AI 出现后。

新增了一种能力：

## “把需求表达给 AI 的能力”

于是很多行业发生变化：

---

## 1. 小团队战斗力暴涨

以前：

一个完整产品需要：

- 前端
- 后端
- 测试
- 运维
- 产品

现在：

AI 能覆盖大量基础工作。

个人开发者时代开始重新出现。

---

## 2. 信息差被压缩

以前很多知识：

靠经验积累。

现在：

AI 可以快速解释：

- 框架
- 文档
- 原理
- 源码

学习速度被极大提升。

---

## 3. “会搜索”正在变成“会提问”

以前：

搜索能力很重要。

现在：

提问能力越来越重要。

因为 AI 更像：

“概率型专家”。

你越会描述问题。

它越容易输出高质量结果。

---

# 十一、但 AI 依然远没有“神化”

虽然 AI 很强。

但目前依然存在大量问题。

例如：

---

## 1. 幻觉仍然存在

尤其：

- 长链推理
- 复杂逻辑
- 数学证明
- 冷门知识

依然可能：

一本正经胡说八道。

---

## 2. 上下文污染

AI 很容易：

被错误上下文带偏。

甚至进入错误循环。

---

## 3. Agent 稳定性问题

很多 AI Agent Demo 看起来很强。

但真实生产环境：

稳定性其实是大问题。

包括：

- Tool 调用失败
- 上下文爆炸
- Token 成本
- 长流程失控
- 死循环

所以 AI Agent 现在本质上：

仍然是“工程问题”。

---

# 十二、我现在对 AI 的看法

现在回头看。

AI 已经不只是：

一个聊天机器人。

而是在逐渐成为：

## “自然语言时代的操作系统”

未来可能会发生：

- AI 操作电脑
- AI 编排工作流
- AI 自动协作
- AI 替代部分重复岗位
- AI 成为人的长期数字助理

但现阶段：

AI 更像：

> “超级副驾驶”

而不是完全自动驾驶。

真正厉害的人。

往往是：

- 懂业务
- 懂工程
- 懂 AI
- 懂如何把问题结构化的人

因为 AI 的本质：

仍然是在放大人的能力。

---

# 十三、现在学 AI 应该学什么？

这是很多人最关心的问题。

如果从工程落地角度。

我认为优先级大概是：

---

## 第一阶段：基础认知

先理解：

- LLM 是什么
- Transformer 是什么
- Token 是什么
- Prompt 是什么
- 上下文是什么
- 幻觉是什么

---

## 第二阶段：AI 应用开发

学习：

- OpenAI API
- Function Calling
- Prompt Engineering
- RAG
- 向量数据库
- Embedding

---

## 第三阶段：Agent 工程化

学习：

- LangChain
- LangGraph
- MCP
- Workflow
- Tool Calling
- 多 Agent

---

## 第四阶段：真实生产落地

学习：

- 成本优化
- Prompt 管理
- 长上下文
- 记忆系统
- 缓存
- 安全
- 权限控制
- 可观测性
- Agent 稳定性

因为：

真正难的已经不是“调用 AI”。

而是：

> “如何让 AI 稳定工作”。

---

# 结尾

这几年 AI 行业的发展速度非常夸张。

很多人甚至感觉：

昨天还只是聊天机器人。

今天已经开始写代码、操作电脑、接管工作流。

而更恐怖的是：

现在可能还只是开始。
